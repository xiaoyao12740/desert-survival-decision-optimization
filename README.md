# Desert Survival Decision Optimization

基于 MATLAB 实现的“穿越沙漠”游戏多情景最优策略规划与决策模型项目。

## 项目简介

本项目围绕数学建模竞赛经典问题“穿越沙漠”展开，研究玩家在资源、天气、路径、收益和时间限制共同作用下的最优决策问题。项目使用 MATLAB 对不同关卡进行建模与求解，分别讨论了：

- 已知未来天气时的最优通关策略
- 仅知道当天天气时的动态决策策略
- 多名玩家同时参与时的协同与竞争策略

项目目标是在满足负重、补给、天气和截止日期等约束条件下，使玩家到达终点后剩余资金尽可能多。

## 研究内容

本项目主要对应以下几类问题：

### 1. 单人、天气已知
在整个游戏周期内，玩家提前知道每一天的天气情况。此时需要综合考虑：

- 是否绕路去矿山挖矿
- 是否经过村庄补给
- 不同天气下行走、停留、挖矿的资源消耗
- 截止日期与负重上限约束

### 2. 单人、天气未知
玩家每天只能知道当天的天气，未来天气未知，因此需要在不确定条件下设计更稳健的行动策略。项目中通过随机抽取天气、类马尔可夫决策建模和蒙特卡洛思想，对第三关和第四关进行求解。

### 3. 多人参与
在多人同时参与时，若玩家在同一区域同步行动、同步挖矿或同步购买资源，会引入额外惩罚，例如：

- 资源消耗加倍
- 挖矿收益平分
- 村庄补给价格升高

因此需要通过错峰行动和路径分离，尽量避免玩家之间的冲突，从而提高整体剩余资金。

## 建模方法

项目中主要使用了以下思想和方法：

- **动态规划**
  - 用于描述多阶段决策过程
  - 逐日更新位置、资源和资金状态

- **全策略枚举与规则推演**
  - 对已知天气场景下的可行方案进行比较
  - 判断是否值得绕路挖矿、是否需要在村庄补给

- **地图简化与路径分析**
  - 将原始地图抽象为关键节点之间的简化路线
  - 降低计算复杂度，便于分析不同方案的优劣

- **随机天气生成**
  - 根据晴朗、高温、沙暴的概率构造天气序列
  - 对未知天气场景进行模拟

- **类马尔可夫决策过程（MDP）**
  - 在未知未来天气时进行状态转移建模
  - 结合即时收益与资源约束确定行动策略

- **多人博弈式策略规避**
  - 针对多人场景设计错峰出发、错峰挖矿和路线规避方案
  - 尽量减少多人同区域导致的额外损耗 :contentReference[oaicite:1]{index=1}

## 项目结果概览

根据论文与代码输出，本项目得到了一组代表性的求解结果：

- **第一关**：最优剩余资金为 **10470 元**
- **第二关**：最优剩余资金为 **12730 元**
- **第三关**：在未知天气条件下，示例最优方案为直奔终点，剩余资金可达 **9350 元**
- **第四关**：在随机天气下，示例最优策略为“起点→村庄→矿山→村庄→矿山→终点”，某次运行结果为 **12310 元**
- **第五关**：双人场景下，采用错峰行动后，两名玩家剩余资金分别可达 **9510 元** 和 **9455 元**
- **第六关**：三人场景下，示例运行中总剩余资金最多可达 **34660 元** 左右

需要说明的是，第三、四、六关由于包含随机天气生成，不同运行结果可能略有差异，但整体策略框架一致。:contentReference[oaicite:2]{index=2}

## 仓库内容说明

根据项目材料，仓库中主要包含以下几类内容：

- 各关卡对应的 MATLAB 求解脚本
- 天气生成与策略推演代码

## 运行环境

- MATLAB
- Excel（用于读取或写入 `Result.xlsx`）

## 使用方法

1. 将项目中的 `.m` 文件与 `Result.xlsx` 放在同一目录下；
2. 在 MATLAB 中打开项目文件夹；
3. 根据需要运行对应关卡的脚本；
4. 程序会在命令行输出策略、消耗、资金结果，并将部分结果写入 `Result.xlsx`。

如果仓库中分为多个脚本，建议按“关卡编号”依次查看和运行。

## 项目特点

- 将游戏规则转化为可计算的最优化问题
- 同时覆盖确定性与不确定性两类情景
- 除单人问题外，还讨论了多人竞争与协同约束
- 代码中保留了大量策略推演细节，便于理解决策逻辑
- 适合作为 MATLAB 数学建模、动态规划与策略模拟案例

## 局限性说明

- 第三、四、六关使用随机天气生成，因此结果带有一定波动性
- 地图在建模时进行了简化，真实地图上的所有细节并未完全保留
- 多人场景的策略仍以规则化分析和可行方案比较为主，尚未完全扩展为更复杂的博弈算法
- 项目更适合作为数学建模与策略分析案例，不是完整商业游戏程序
